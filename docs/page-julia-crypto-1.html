暗号理論

AとB = Alice と Bob

暗号の人たちは、AとかBとかにアリスとかボブとかいう名前を付けたがります。 大体、以下のような感じになっていると考えて下さい。

名前 name 詳細
アリス Alice 一般人
ボブ Bob 一般人
キャサリン Cathy 一般人
デイブ Dave 一般人
イブ Eve 盗聴者(eavesdropper?)。通信を盗聴する。
\(\vdots\) \(\vdots\) \(\vdots\)
マロリー Mallory 能動的攻撃者(malicious？)。通信妨害やメッセージの偽装をする。
\(\vdots\) \(\vdots\) \(\vdots\)
ヴィクター Victor 検証者(validator?)。
\(\vdots\) \(\vdots\) \(\vdots\)

例えば、アリスがボブにメールを送るとき、悪意のある盗聴者イブがメールを盗み見ようとしたりします。 暗号理論では悪意のある第三者の存在を前提にすることが多いです。

暗号化(encrypt)と復号化(decrypt)

暗号化とは、平文(plaintext)を、意味の分からない文、つまり暗号文(ciphertext)に変換することです。 復号化はその逆で、暗号文から平文に変換することです。

普通、平文よりも暗号文の方がサイズの大きいデータになります。

flowchart LR
  a["平文=Hello World!"] -- 暗号化 -->b["暗号文=0avjmgc,qn7%xa#mx"]
  b -- 復号化 --> a

送信者、受信者、盗聴者

インターネット上でやり取りされるデータは、通信路の間でコピーすることが容易です。 そのため、以下の図におけるEveのような盗聴者が存在するものとして考える必要があります。

flowchart LR
  a2 --> b2
  a2 --> c2
  subgraph Alice_送信者
    a1[平文] -- 暗号化 --> a2[暗号文]
  end
  subgraph Bob_受信者
    b2[暗号文] -- 復号 --> b1[平文]
  end
  subgraph Eve_盗聴者
    c2[暗号文]
  end
