数値解析の基礎

QR分解

QR分解（QR decomposition）は、行列 \(A\) を直交行列 \(Q\) と上三角行列 \(R\) の積に分解する手法です。

QR分解は数値解析や線形代数の応用分野で広く使われています。

\[ A = QR \]

QR分解は正方行列に限らず、 \(A \in \mathbb{R}^{m\times n}, m \ge n\) の行列に適用できます。その時の式は以下の通りです。

この場合も \(Q \in \mathbb{R}^{m \times m}, R \in \mathbb{R}^{m \times n}\) の形式を使うのが一般的ですが、数値計算上の効率性を考えると \(Q_1 \in \mathbb{R}^{m \times n}, R_1 \in \mathbb{R}^{n \times n}\) の形式（エコノミー分解形式）を使う方がより適切です。

\[ A = Q R = \begin{bmatrix} Q_1 & Q_2 \end{bmatrix} \begin{bmatrix} R_1 \\ O \end{bmatrix} = Q_1 R_1 \]

グラム・シュミットの直交化法

「グラム・シュミットの直交化法によるQR分解」は、QR分解を分かりやすく説明するうえで重要なアルゴリズムです。

ただし、実際の実装では使いにくいため、このアルゴリズムを用いて数値計算を行うことは稀です。

graph LR
  A[A] -- 直交化 --> B[Q]
  B --> C[R]

\(A\) からグラム・シュミットの直交化により、\(Q\) を構築

グラム・シュミットの直交化は、ベクトル \(\{x_j \in \mathbb{R}^{m}\}_{j=1}^{n}\) を使って直交する \(n\) 本のベクトル \(\{e_j \in \mathbb{R}^{m}\}_{j=1}^{n}\) を作るアルゴリズムです。

この手法は、次のような漸化式で計算されます(\(e_j\) などは縦ベクトルを想定)。

これをプログラムで実装することを考える。

\[ e_i = \frac{u_i}{\|u_i\|}, \;\;\; u_i = x_i - \sum_{j=1}^{i-1} e_j e_j^\mathrm{T} x_i, \;\;\; e_1 = \frac{x_1}{\|x_1\|} \]

\(n\) についての漸化式によって、\(e_i\) を求めるプログラムを実装すると、以下の通りになる。

using LinearAlgebra

function e_orth!(x, E, i)
    u = x - sum( E[:,j]' * x * E[:,j] for j = 1:i-1 )
    e = u / norm(u)
    E[:,i] = e
end
;

よって、グラム・シュミットの直交化は以下のように実装される。

function orth_gram_schmidt(X)
   E = zeros(size(X))
   E[:, 1] = X[:,1] / norm(X[:,1])
   for i = 2:size(X, 2)
       xi = X[:,i]
       e_orth!(xi, E, i)
   end
   return E
end

E = orth_gram_schmidt(rand(3, 3))
E' * E # 単位行列になっている。

\(Q\) から \(R\) を構築

作成した直交行列を用いると、行列 \(A=[a_1, a_2, ..., a_n]\) の各ベクトル \(a_i\) は以下のように表現されることになります。

\[ a_i = \sum_{j=1}^{i} e_j e_j' a_i = [e_1, \dots, e_i, e_{i+1}, \dots, e_n] \begin{bmatrix}e_1' a_i \\ \vdots \\ e_i' a_i \\ 0 \\ \vdots \\ 0 \end{bmatrix} \]

よって、\(A\) は以下のようになります。

\[ A = \begin{bmatrix} e_1'a_1 & e_1'a_2 & e_1'a_3 & \dots \\ 0 & e_2'a_2 & e_2'a_3 & \dots \\ 0 & 0 & e_2'a_3 & \dots \\ \vdots & \vdots & \vdots & \vdots \end{bmatrix} \]

これをプログラムで実装すると、以下の通りになります。

function Q_to_R(Q, A)
    n = size(A, 2)
    R = zeros(n, n)
    for i = 1:n
        for j = i:n
            ei, aj = Q[:,i],  A[:,j]
            R[i,j] = ei' * aj
        end
    end
    return R
end

A = rand(5, 4)
Q = orth_gram_schmidt(A)
R = Q_to_R(Q, A)

QR分解の実装

グラム・シュミットの直交化法によりQR分解を行う関数myqr_gram_schmidtは以下の通りになります。

function myqr_gram_schmidt(A)
    Q = orth_gram_schmidt(A)
    R = Q_to_R(Q, A)
    return Q, R
end

A = rand(6, 4)
Q, R = myqr_gram_schmidt(A)

norm(A - Q*R) 
6.562392647709663e-16

ハウスホルダー法

「ハウスホルダー法によるQR分解」は、QR分解の実用的なアルゴリズムの１つです。

graph LR
  A[A] --> B[A']
  B -- 変換 --> C[Qi]
  C -- 小行列化 --> B
  C --> D[R]

ハウスホルダー変換の意味

ハウスホルダー変換とは、法線ベクトル \(e\) の超平面に対して対称に点を写す変換のことです。

変換を表す以下の行列 \(H\) は直交行列かつ対称行列です。

\[ H = I - 2 e e^\mathrm{T} \]

ハウスホルダー変換は直交変換であるため、どんな超平面を選んでも元のベクトルのノルムを半径とした超球面上にしか変換を行うことはできないことに注意します。

そうすると、例えば、以下のような変換を行うように \(e\) を選んでやることもできます。

\[ y = Hx, \;\;\; y = \begin{bmatrix} \|x\| \\ 0_{m-1} \end{bmatrix} \]

このときの \(e\) は以下のベクトルになる。

\[ e =\frac{x-y}{\|x-y\|} \]

QR法でのハウスホルダー変換

以下、全体の変換により得られる行列を \(A_i\) 、行列Aを変換していく際の小行列として表れる行列を \(B_i\) と定めます。

\[ A_1 = A = B_1 \]

行列 \(A_1 = A=[a_!, \dots , a_n]\) に対して、 \(x=a_1, \;\;\; y=[\|x\|, 0_{m-1}^\mathrm{T}]^\mathrm{T}\) とした時のハウスホルダー変換行列 \(H_1\) を作用させると以下のようになります。

\[ A_2 = H_1 A_1 = [H_1 a_1 , \dots , H_1 a_n] = \begin{bmatrix} \|a_1\| & * \\ 0_{m-1} & B_2 \end{bmatrix} \]

ここで、小行列を逐次作用させていくことで、\(k\) 次上三角行列 \(R_k\) と小行列 \(B_k\) により、以下のような行列が作れるものとする。

\[ A_k = \begin{bmatrix} R_k & * \\ O & B_{k} \end{bmatrix} \]

ここで、\(B_k\) という行列に対しても同じようにハウスホルダー変換を施すことができるので、その行列を \(H_k\) とすると、以下のように、\(A_k\) から \(A_{k+1}\) を得る式が求められます。

\[ \begin{bmatrix} I & O^\mathrm{T}\\ O & H_{k} \end{bmatrix} A_k = \begin{bmatrix} I & O^\mathrm{T}\\ O & H_{k} \end{bmatrix} \begin{bmatrix} R_k & * \\ O & B_{k} \end{bmatrix} = \begin{bmatrix} R_k & *\\ O & H_{k}B_{k} \end{bmatrix} = \begin{bmatrix} R_{k+1} & * \\ O & B_{k+1} \end{bmatrix} = A_{k+1} \]

QR法でのハウスホルダー変換の漸化式

まとめると、\(Q\)、\(R\) を構築する以下の漸化式が得られます。

\[ R = A_n, \;\;\; Q = Q_1^\mathrm{T} Q_2^\mathrm{T} \dots Q_t^\mathrm{T}, \;\;\; t=\text{min}(m-1, n) \]

\[ A_{k+1} = Q_k A_k, \;\;\; A_k=\begin{bmatrix} R_k & * \\ O & B_k \end{bmatrix} \;\;\; A_1 = A = B_1 \]

ここで、直行行列 \(Q_k\) は以下の式で表されます。 なお、\(I\) は \(k\) 次単位行列、\(O\) はそれぞれの大きさのゼロ行列です。

\[ Q_k = \begin{bmatrix} I & O^\mathrm{T}\\ O & H_{k} \end{bmatrix}, \;\;\; H_k = I - 2 ee', \;\;\; e =\frac{x-y}{\|x-y\|}, \;\;\; x = b_1, \;\;\; y=\begin{bmatrix} \|b_1\| \\ 0 \end{bmatrix}, \;\;\; b_1 = B_k[:, 1] \]

QR分解の実装

ハウスホルダー変換を行う行列を生成するプログラムは以下の通りです。

using LinearAlgebra

# e を使って行列 H_k を構築する。
H(e)    = Matrix{Float64}(I, length(e), length(e)) - 2 * e * e'

# x, y を使って行列 H_k を構築する。
H(x, y) = H((v -> v / norm(v))(x - y))

# b1 を使って行列 H_k を構築する。
function Bk_to_Hk(Bk)
    b1 = Bk[:,1]
    x  = b1
    y  = zeros(size(b1)); y[1] = norm(b1)
    return H(x, y)
end

# 行列 Q_k を構築する。
function Ak_to_Qk!(Qk, Ak, k)
    Qk[:, :] .= 0
    Qk[1:k, 1:k]     = Matrix{Float64}(I, k, k)
    Qk[k:end, k:end] = Bk_to_Hk(Ak[k:end, k:end])
end
;

以下のように、本来計算上0になるべき部分にごく小さい値が入っているため、効率はやや悪いですが、細かい部分なのでそこを気にするのは取り敢えず止めておきます。

漸化式を適用していく部分を実装すると以下の通りになります。

function A_next!(Q, Qk, Ak, k)
    Ak_to_Qk!(Qk, Ak, k)
    Ak .= Qk*Ak 
    Q  .= Q*Qk'
end
;

QR分解の実装は以下の通りになります。

function myqr_house(AA)
    m, n  = size(AA)
    Ak = copy(AA)
    Qk = zeros(m, m)
    Q  = Matrix{Float64}(I, m, m)
    t = min(m-1, n)
    for k = 1:t
        A_next!(Q, Qk, Ak, k)
    end
    R = Ak
    return Q, R
end

A = rand(4, 4)
Q, R = myqr_house(A)

norm(A - Q*R)
6.959765930514777e-16

ギブンス回転

「ギブンス回転」を用いる方法は、QR分解の実用的なアルゴリズムの１つです。 並列化に向いている手法です。

説明は暇があったら書きます。

# 行列の下三角の部分だけを返す
function tril_indices(m, n)
    idx = []
    for i = 1:m
        for j = 1:i-1
            if j > n
                break
            end
            push!(idx, (i,j))
        end
    end
    return idx
end
;
function cos_givens_rotation(a, b)
    r = √(a^2 + b^2)
    rot = [
        a/r -b/r; #  cos sin
        b/r  a/r  # -sin cos
    ]
    return rot
end


function myqr_givens_rotation(A)
    m, n = size(A)
    Q = Matrix{Float64}(I, m, m)
    R = copy(A)
    for ij = tril_indices(m, n)
        i, j = ij
        if R[i, j] == 0
            continue
        end
        G = Matrix{Float64}(I, m, m)
        G[[j, i], [j, i]] = cos_givens_rotation(R[j, j], R[i, j])

        R = G * R
        Q = Q * G'
    end
    return Q, R
end
;
A = rand(6, 4)
Q, R = myqr_givens_rotation(A)

norm(A - Q*R)
4.965068306494546e-16
