Juliaのマクロ

マクロとは

マクロとは、コンパイル時に実行されるコード生成機能であり、柔軟なコード記述と静的なコード変換を可能にします。

マクロはソースコードのAST（抽象構文木）に対して操作を行い、新しいコードを生成します。

Juliaのマクロは、@を使って呼び出せます。

呼び出しは、マクロに渡される引数を含んだ式として解釈され、マクロ内の処理が実行されます。その後、マクロは評価結果として新しい式を返します。この返された式は、通常のコードとして実行されます。

つまり、マクロはプログラムを生成するプログラムと言えます。

マクロは、繰り返しパターンやコードの自動生成、ドメイン固有言語（DSL）の構築など、柔軟なコード記述を可能とします。 マクロは、静的なコード解析やコード変換を行うための強力なツールです。

マクロの記述方法

マクロは以下のコードで定義されます。

macro マクロ名(引数1, 引数2, ...)
  処理内容
  return 式
end

マクロの使用方法は以下のように@を使えばよいです。

@マクロ名 引数1 引数2 ...

クオート（引用）

クオートは以下のどちらかの方法で行われます。 なお、expr1, expr2は、そのものがプログラムだと考えてください。

:(expr)を使う方法

expr1 = :(x + 1)
:(x + 1)

quote(), quote ~ endを使う方法

expr2 = quote 
    x + 1
end
quote
    #= In[3]:2 =#
    x + 1
end

アンクオート（引用解除）

quote, :()でクオートした式（プログラム）の内部でアンクオート$という動作を行わせることができます。 アンクオートした部分の式(以下の$(y+1))は、マクロの展開時に評価されます。

y = 3 # マクロ展開時
expr = quote 
    z = $(y + 1) + y
end
display(expr)
quote
    #= In[4]:3 =#
    z = 4 + y
end

評価

クオートした式を実行して評価するには、evalを使ってください。 例えば、上で作成した \(z \gets y+1\) を表現する式exprは、以下のように評価できます。

y = 1
eval(expr)
display(z)
5

ASTの生成

AST(抽象構文木)を作成するには、以下のようにdumpが使えます。

expr = :(x + y / z)
dump(expr)

  args: Array{Any
}((3,))

    2: Symbol x
    3: Expr

エスケープ

マクロの展開時に式が評価されることから保護するため、escが使えます。

これを使わないと、マクロを展開するときに勝手に式が評価されてしまいます。

なので、以下のように、式をエスケープしてコードに埋め込んでからアンクオートすることでプログラムの順序が守られます。

macro mymacro(expr)
    quote 
        println("-"^10)
        $(esc(expr))
        println("-"^10)
    end
end

name = "YAMADA"
@mymacro println("Hello ", name)
----------
Hello YAMADA
----------

マクロの例

時間計測のマクロは以下の通りです。 実のところ、@timeとか@timedなどのマクロが既にあるので、あまり意味はないのですが……。

macro time_measure(expr)
    quote
        local start_time = time()
        local result = $(esc(expr))
        local end_time = time()
        println("Execution time: ", end_time - start_time, " seconds")
        result
    end
end

A = @time_measure begin
    sleep(0.05)
end
;
Execution time: 0.0559999942779541 seconds
