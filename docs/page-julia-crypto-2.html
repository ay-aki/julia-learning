暗号理論

暗号アルゴリズム（計算機登場以前）

  • シーザー暗号

    シーザー暗号は、古代ローマの将軍であるガイウス・ユリウス・カエサルが使用したとされる、非常にシンプルな代替暗号の一種です。 シーザー暗号は、アルファベットの文字を一定の数だけシフトさせて暗号化します。

  • 単位換字暗号

    単一換字暗号は、平文の各文字を別の文字に1対1で置き換える暗号化手法です。

  • エニグマ

    エニグマは、第二次世界大戦中にドイツが使用した複雑な電子機械式暗号装置です。

シーザー暗号

シーザー暗号は、「鍵」を文字のシフト数 \(n \in K=\{0,1,2,\dots,2\}\) とする暗号です。 鍵空間 \(K\) の大きさはここでは \(|K|=26\) であり、ブルート・フォース・アタックでの突破が容易な暗号である。

文字のシフトとは、例えば以下のように、表をもとに文字をずらす手法です。

'あ' + 5
'ぇ': Unicode U+3047 (category Lo: Letter, other)

それぞれの文字（アルファベット）を n_shift だけシフトさせるプログラムは以下の通りです。

function char_shift(c, n_shift)
    if !isletter(c)
        return c
    end
    if !('a' <= lowercase(c) <= 'z')
        return c
    end
    a = 'a'
    if isuppercase(c)
        a = 'A'
    end
    i = c - a
    j = (i + n_shift) % 26
    return a + j
end

char_shift.(['a', 'X'], 3)
2-element Vector{Char}:
 'd': ASCII/Unicode U+0064 (category Ll: Letter, lowercase)
 'A': ASCII/Unicode U+0041 (category Lu: Letter, uppercase)

シーザー暗号の実装は以下の通りです。

function encode_caesar(
    plaintext::AbstractString, 
    n_shift::Integer # key
)
    l_plaintext  = collect(plaintext)
    l_ciphertext = char_shift.(l_plaintext, n_shift)
    ciphertext   = join(l_ciphertext)
    return ciphertext
end

function decode_caesar(ciphertext, n_shift)
    return encode_caesar(ciphertext, 26 - n_shift)
end

# key
mykey = 3

# encode
plaintext  = "hello world"
ciphertext = encode_caesar(plaintext, mykey)

# decode
plaintext_ = decode_caesar(ciphertext, mykey)

plaintext_
"hello world"

単一換字暗号

単一換字暗号は、「鍵」を換字表 \(\text{Tbl} \in K\) とする暗号です。 鍵空間 \(K\) の大きさは \(|K|=26!=403291461126605635584000000\) であり、ブルート・フォース・アタックでの突破は困難ですが、頻度分析により比較的簡単に突破できます。

それぞれの文字（アルファベット）を Tbl に従って置換するプログラムは以下の通りです。

function char_replace(c, tbl)
    if !isletter(c)
        return c
    end
    if !('a' <= lowercase(c) <= 'z')
        return c
    end
    return tbl[c]
end
char_replace (generic function with 1 method)

単一換字暗号の暗号は以下の通りです。

function encode_monoalphabetic(
    plaintext::AbstractString, 
    key
)
    tbl = Dict(
        zip(
            collect(key[1]), collect(key[2])
        )
    )
    l_plaintext  = collect(plaintext)
    l_ciphertext = (c -> char_replace(c, tbl)).(l_plaintext)
    ciphertext   = join(l_ciphertext)
    return ciphertext
end

function decode_monoalphabetic(ciphertext, key)
    key = (key[2], key[1])
    return encode_monoalphabetic(ciphertext, key)
end

# key
mykey = (
    collect("abcdefghijklmnopqrstuvwxyz"), 
    collect("gpkhayvwcoxzqntudeflmijbrs")
)

# encode
plaintext  = "hello world"
ciphertext = encode_monoalphabetic(plaintext, mykey)

# decode
plaintext_ = decode_monoalphabetic(ciphertext, mykey)

plaintext_
"hello world"

エニグマ

エニグマ（Enigma）は、第二次世界大戦中にドイツで使用された暗号化機械です。 当時は非常に高度な暗号化手法であるとされ、暗号文の解読が非常に困難であると考えられていました。

機械式の代替暗号装置であり、電気信号を用いて文字を暗号化した暗号です。

エニグマは、複数の可変ローター（回転ディスク）とプラグボード（Steckerboard）と呼ばれる接続装置から構成されており、これによって文字の置換と交換が行われました。 暗号学の歴史上、重要な節目となる装置であり、暗号解読の研究や教育においても広く取り上げられています。 平文の文字を入力すると、可変ローターの回転と接続装置の置換によって暗号文の文字が出力されます。

エニグマは KEK(key encrypting key)と言われる、鍵を暗号化するための日替わり鍵を導入したことでも知られています。 そのため、同じ文字が連続して入力されても、異なる文字が出力されるため、エニグマは多重換字暗号として知られる複雑な暗号方式でした。

解読アルゴリズム

  • ブルート・フォース・アタック

    ブルート・フォース・アタックは、全ての可能な組み合わせを試行することで、暗号を解読する攻撃手法です。

  • 頻度分析

    頻度分析は、暗号文の中の文字や記号の出現頻度を解析し、元の平文を特定する手法です。

古典的な頻度分析の手法として、以下のようなアイデアがある。文字列の中に頻出する文字（文字列）を推定できる可能性がある。ただし、これらのアイデアは偏りのない巨大な文章に対しては意味があるが、そうでなければ上手くいかない。

  • 出現するアルファベットの頻度

    多い順に、e, t, a, o, i, n, …, j, x, q, z

  • 出現する２文字のアルファベット文字列の頻度

    多い順に、th, he, in, er, …

  • 出現する３文字のアルファベット文字列の頻度

    the, and, ing, ion,

    参考

これらのアイデアを利用するために、以下のようにパターンを解析するプログラムを実装した。

function pattern_frequency_analysis(
    text::AbstractString, n::Integer
)
    pattern_counts = Dict() # Dict{SubString, Int64}()
    for i in 1:length(text)- n + 1
        pattern = text[i:i+n-1]
        if haskey(pattern_counts, pattern)
            pattern_counts[pattern] += 1
        else
            pattern_counts[pattern] = 1
        end
    end
    result = sort(collect(pattern_counts), by = x -> x[2], rev=true)
    return result
end

function frequency_analysis(ciphertext, N)
    for n = 1:N
        result = pattern_frequency_analysis(ciphertext, n)
        display(result[1:8])
    end
end
